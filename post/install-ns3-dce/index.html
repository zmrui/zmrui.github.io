Install NS3 DCE on Ubuntu 16.04 in Dec. 2021

Since the NS3-DCE only tested/ works on Ubuntu 16.04  https://www.nsnam.org/docs/dce/manual/html/intro.html

GCC problem

However, the gcc-5.4 on Ubuntu 16.04 does not meet the requirement of the waf script. So, directly executing the code on Quick Start  will be failed and without bake.py output with flag -vvv.

One should update the GCC version to minimal 8.0 before build NS3 and NS3-dce by using

sudo add-apt-repository ppa:ubuntu-toolchain-r/test
sudo apt-get update
sudo apt-get install -y gcc-8 g++-8
sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-8 8 --slave /usr/bin/g++ g++ /usr/bin/g++-8
gcc -v

NS3-DCE-version problem

While the manual says that the ./bake.py configure -e dce-ns3-version can be point as dce-ns3-dev, however, only dce-ns3-1.11 works since the dce-ns3-dev may compile two ns3 versions(ns3-dev and ns3-3.34 ) and force the ns3-dce build was script to stop.

So, just use dce-ns3-1.11:

./bake.py configure -e dce-ns3-1.11

If this config failed with

>>> Error: missing Python 'distro' package; please install

Install python3-pip first, and use pip to install distro by:


sudo apt install python3-pip
pip instal distro

Remember no matter what pip encourage to update pip, keep pip as installed from apt source, as pip’s new version does not support python3.5.

Libc not-detect problem

The bake.py does not detect libc6 file, and I found this can be solved by installing i386 libc6 from apt.

Using

sudo apt-get install libc6:i386

Full flow works in Dec. 2021, Ubuntu 16.04

sudo apt update 
sudo apt install git

sudo add-apt-repository ppa:ubuntu-toolchain-r/test 
sudo apt-get update 
sudo apt-get install -y gcc-8 g++-8 
sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-8 8 --slave /usr/bin/g++ g++ /usr/bin/g++-8

git clone https://gitlab.com/nsnam/bake.git
cd bake
export PATH=$PATH:`pwd`/build/bin:`pwd`/build/bin_dce
export PYTHONPATH=`pwd`/build/lib${PYTHONPATH:+:$PYTHONPATH}
export LD_LIBRARY_PATH=`pwd`/build/lib${LD_LIBRARY_PATH:+:$LD_LIBRARY_PATH}
export DCE_PATH=`pwd`/build/bin_dce:`pwd`/build/sbin

sudo apt-get install -y python3-pip
pip3 install distro

./bake.py configure -e dce-ns3-1.11
./bake.py check
./bake.py show

#Install missing packeges
sudo apt install cmake
sudo add-apt-repository -y ppa:mercurial-ppa/releases
sudo apt-get update
sudo apt-get install -y python3-dev
sudo pip3 install mercurial --upgrade

sudo apt-get install qt5-default
sudo apt-get install python3-gi-cairo
sudo apt-get install libpcap-dev
sudo apt-get install python3-gi python3-pygraphviz
sudo apt-get install libc6:i386

#Build
./bake.py download
./bake.py build

Even the gi-cairo, pygobject,  pygraphviz were not available, building ns3 and ns3-dce still succeed.
